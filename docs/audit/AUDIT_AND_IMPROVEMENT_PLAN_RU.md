# Открытый план улучшений (аудит)

Связанные документы: [PROJECT_AUDIT_2026_04_RU.md](./PROJECT_AUDIT_2026_04_RU.md) · [DEPLOY_READINESS_2026_04_RU.md](./DEPLOY_READINESS_2026_04_RU.md) · [QUALITY_GATES_SYNC_RU.md](./QUALITY_GATES_SYNC_RU.md) · [`premier-design/CHANGELOG.md`](../../premier-design/CHANGELOG.md)

## Статус

На текущий цикл **открытых задач по аудиту нет**: последние пункты по карточкам/контрасту dark-темы/фокусу, формам и визуальному smoke-контролю закрыты и синхронизированы в `CHANGELOG.md`.

## Definition of Done для новых задач аудита

Задача считается закрытой только если:
- есть реальное изменение в коде или CI-процессе (не только текстовая пометка);
- обновлены релевантные документы: `PROJECT_AUDIT_2026_04_RU.md` и/или `DEPLOY_READINESS_2026_04_RU.md`;
- есть запись в `premier-design/CHANGELOG.md`;
- пройдены необходимые проверки (`yarn lint`, при изменении поведения — тесты/сборка/смоки).

## Как добавлять новую задачу

Добавляйте строку в формате:

| ID | Приоритет | Задача | Артефакт закрытия | Статус |
|----|-----------|--------|-------------------|--------|
| UI-XX | P1/P2/P3 | Краткая формулировка | PR/файлы/команды | Open/In progress/Done |

> Исторические выполненные пункты хранятся в `PROJECT_AUDIT_2026_04_RU.md` (блок «Закрытые направления») и `CHANGELOG.md`.
