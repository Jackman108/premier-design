# Feature Structure Roadmap (RU)

## Цель

Привести все модули `features/*` к единому шаблону:

- `ui/`
- `hooks/`
- `interface/`
- `utils/`
- `__tests__/` (по мере критичности сценария)

## Текущий механизм контроля

- Скрипт: `scripts/check-feature-structure.mjs` (корень репозитория)
- Команда: `yarn check:feature-structure`
- Интеграция: CI + `lint-staged`

## Принцип внедрения

1. Не ломать существующие фичи массовым переносом за один PR.
2. Мигрировать поэтапно:
    - сначала новые и активно изменяемые фичи,
    - затем legacy-модули.
3. Не создавать "пустые" каталоги с `.gitkeep` ради прохождения проверок.
4. Если для фичи пока нет отдельного `hooks/utils`, слой не форсируется искусственно: добавляется только при реальной логике.

## Критерий завершения

- `check-feature-structure` проверяет обязательное наличие **`ui/`** и **`interface/`** у каждой фичи (без пустых `hooks/` / `utils/` ради CI).
- Новые PR не добавляют модулей вне шаблона; по мере появления логики — вынос в `hooks/` и `utils/` по правилам слоя `ui`.
