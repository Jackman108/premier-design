# Зависимости: только Yarn

## Правило

- **Единственный lock-файл в git:** корневой `yarn.lock` (рядом с `package.json`).
- **`package-lock.json` не коммитим** (игнор в корневом `.gitignore`). Если npm создал файл локально — удалите или не добавляйте в коммит.
- Установка и CI: **`yarn install`**, **`yarn install --frozen-lockfile`** в пайплайнах.

## Аудит уязвимостей

- В CI: `yarn audit --level critical` (см. `.github/workflows/ci.yml`). **Yarn v1:** при любой находке в дереве команда завершится с **ненулевым кодом** (часто `4`), даже если в таблице остались только уровни ниже порога вывода; `--level` в первую очередь задаёт **фильтр печати**. Для зелёного гейта нужен чистый аудит по lockfile (при необходимости — `resolutions` в `package.json`).
- Локально перед релизом: `yarn audit` (полная картина) или с `--level` для сужения вывода.

## Не использовать

- Скрипт `npm install --package-lock-only` и хранение `package-lock.json` в репозитории — снято; не возвращать без отдельного ADR.
