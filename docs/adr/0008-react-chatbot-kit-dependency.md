# ADR 0008: зависимость `react-chatbot-kit`

**Статус:** принято (ограничения зафиксированы)  
**Дата:** 2026-04-17

## Контекст

Виджет чата использует `react-chatbot-kit`. Версия в экосистеме обновляется редко; возможны задержки с совместимостью React и отсутствием исправлений по CVE относительно крупных вендоров.

## Решение

1. **Оставить** библиотеку до появления продукта-обоснования замены (стоимость миграции vs. ценность сценария).
2. **Изоляция бандла:** логика и конфиг чата не импортируются из модулей, которые грузятся до открытия чата; рантайм (`ChatBotRuntime`) подключается только через `dynamic(..., { ssr: false })` при открытой панели.
3. **Контроль:** при обновлении зависимостей — `yarn audit`, smoke чата; при критичном CVE — оценка форка, обёртки или замены на лёгкий самописный UI поверх того же сценария (use-case без привязки к бренду библиотеки).

## Последствия

- Смена библиотеки затронет только слой `features/buttons-panel` и конфиг/провайдеры сообщений.
- Документированный риск снижается за счёт ленивой загрузки и узкой поверхности импорта.

## План изоляции/замены (S4, PERF-03)

1. **Триггер на замену:** рост initial JS выше budget или high/critical CVE без патча в течение 1 релизного цикла.
2. **Фаза 1 (изоляция):** зафиксировать единый runtime-адаптер в `features/buttons-panel`, чтобы UI-контракт чата не зависел от конкретной библиотеки.
3. **Фаза 2 (parallel spike):** прототип лёгкой замены (headless UI + текущий сценарий диалога) в отдельном RFC/ветке без переключения prod.
4. **Фаза 3 (migration gate):** замена только после сравнения bundle delta, smoke e2e и отсутствия regressions по lead funnel.
5. **Rollback:** фича-флаг/быстрый откат на текущий runtime до закрытия регрессий.

Детализация этапов и критериев — в [`docs/audit/rfc-chatbot-runtime-isolation-ru.md`](../audit/rfc-chatbot-runtime-isolation-ru.md).

## Проверка готовности

- В чанке первой загрузки панели нет прямого импорта `react-chatbot-kit` из хуков, общих для SSR/CSR без открытия чата.
