# ADR 0011 — Проприетарная лицензия Premier Design

- **Статус:** Принято (2026-04-27)
- **Тип решения:** правовое / интеллектуальная собственность

## Контекст

Репозиторий хостит коммерческий продукт ИП Коробов Е.В. (бренд Premium Design).
До этого решения в `package.json` стояло только `"license": "UNLICENSED"`, и в репозитории
не было файла `LICENSE` — формально это означает «все права защищены», но не даёт чёткой
рамки сотрудникам и подрядчикам и не обозначает условия использования.

Цели:
- зафиксировать исключительные права Правообладателя на исходный код, дизайн, копирайтинг;
- разрешить внутренний доступ контрибьюторам/подрядчикам в рамках их обязательств;
- закрыть форки и копии без письменного разрешения;
- учитывать, что часть зависимостей — open-source (их лицензии не релицензируют наш код).

## Решение

1. В корне репозитория публикуются **два** файла лицензии:
   - [`LICENSE`](../../LICENSE) — английская версия (proprietary "All Rights Reserved"),
     для распознавания GitHub и международного контекста;
   - [`LICENSE_RU.md`](../../LICENSE_RU.md) — русская версия, юридически релевантная для РБ;
     при расхождении в РБ преобладает русская версия.
2. В `premier-design/package.json` поле `license` меняется на
   `"SEE LICENSE IN ../LICENSE"` (стандартное значение npm для проприетарного кода)
   и заполняется поле `author`.
3. Сторонние компоненты остаются под своими лицензиями (см. `package.json`/`yarn.lock`/SBOM)
   и не релицензируют проект.
4. Запросы по лицензированию направляются на `admin@premier-design.pro`
   (контакт совпадает с `SITE_OPERATOR.privacyContactEmail`).

## Последствия

- Любой клон/форк репозитория без письменного разрешения формально нарушает лицензию.
- В CI/CD ничего менять не требуется: образы Docker и приватные репозитории и так не
  публикуются вовне.
- При найме новых разработчиков — упоминать раздел «Permitted Internal Use» в договорах.
- При подключении open-source зависимостей продолжаем уважать их лицензии и не оборачиваем
  их в проприетарные ограничения.

## Проверка готовности

- [x] `LICENSE` и `LICENSE_RU.md` существуют в корне;
- [x] `package.json` указывает на `LICENSE`;
- [x] `docs/README.md` ссылается на лицензию;
- [x] [`changelog.md`](../changelog.md) содержит запись о принятии ADR.
