# ADR 0007: Pages Router и App Router (Next.js) — архив

**Статус:** заменено  
**Дата:** 2026-04-17 · актуализация структуры 2026-04-30

## Контекст (исторический)

Ранее в репозитории обсуждался гибрид **Pages Router** (`pages/`) и **App Router** (`app/`): часть URL на `pages/`, юридические страницы и изолированные зоны — в `app/`.

## Текущее состояние

Маршрутизация на **едином App Router** в **`src/app/`**. Каталог **`pages/`** (Pages Router) **не используется**. Композиция страниц — **`src/pages-layer/`**, публичный API слоя страниц — алиасы **`@pages`**, **`@pages-layer/*`**. Route Handlers — **`src/app/api/**`\*\*.

Структура репозитория выровнена с **febcode**: **`package.json`** и конфиги сборки в **корне** клона, исходники приложения в **`src/`**.

## Прежние принципы (архив)

1. Не смешивать один и тот же публичный URL между двумя деревьями маршрутизации.
2. Документы и зоны с отдельным UX — в `app/` при сохранении паритета токенов/виджетов.

Историческая миграция на полный App Router зафиксирована в **`docs/changelog.md`** (PD-R-07 и связанные записи).

## Проверка готовности (актуально)

- Новый маршрут добавляется только под **`src/app/`** или через композицию в **`src/pages-layer/`**, без конфликта с зарезервированными путями.
- SEO: `generateMetadata`, JSON-LD — по правилам текущего стека (см. гайды по слоям и SEO).
