# ADR 0004: Ужесточение CSP и доверенные SVG (`next/image`)

## Статус

Принято (апрель 2026).

## Контекст

Аудит (§5.1–5.4) зафиксировал:

- широкий `img-src` с произвольным `https:`;
- широкий `connect-src` с произвольным `https:` в production;
- `dangerouslyAllowSVG: true` без явного контракта доверия;
- `script-src 'unsafe-inline'` — типичное ограничение для Next.js Pages до внедрения nonce.

## Решение

1. **`img-src`**: только `'self'`, `data:`, `blob:` и хосты Google Maps, используемые картой и встраиваемым контентом (`www.google.com`, `maps.gstatic.com`, `maps.googleapis.com`).
2. **`connect-src` (production)**: только `'self'` — клиентский код обращается к собственному API и `dataLayer`; вызовы Telegram выполняются на сервере и не попадают в браузерный CSP.
3. **`script-src`**: в production сохранён `'unsafe-inline'` из‑за сгенерированных скриптов Next.js; дальнейшее сужение — через **nonce** в `middleware` + прокидывание в документ (отдельная задача).
4. **`next/images`**: `dangerouslyAllowSVG: true` оставлен **только** при контракте «SVG исключительно из репозитория (`/public`, импорты из `public/*`)»; для оптимизатора задан минимальный CSP с `sandbox` без `default-src 'self'` для встроенного контекста превью.

## Последствия

- При появлении легитимных внешних изображений нужно явно добавить домен в `img-src` в `next.config.js`.
- При подключении аналитики с внешними endpoint (GA4, YM) — расширить `connect-src` точечно.

## Проверка готовности

- `yarn build`, smoke главной и страницы с картой контактов.
- При ошибках CSP в консоли браузера — дополнить список доменов или зафиксировать исключение в этом ADR.
