# ADR 0001: UI-стек, дизайн-токены и путь к библиотеке компонентов

## Статус

Принято (2026-04-15)

## Контекст

- Сайт уже использует **React + CSS Modules + глобальные CSS-переменные** в `styles/globals.css`.
- В исходных требованиях фигурировал **NextUI**, но в кодовой базе он **не подключён**; массовая миграция экранов дала бы высокий риск регрессий и простой поставки.
- Нужен единый контракт для новых экранов и для агентских/человеческих изменений без смешения стилей «вручную» в каждом модуле.

## Решение

1. **Базовый UI-стек (зафиксирован):** существующие **CSS Modules + токены**; новые примитивы размещаются в `shared/ui/primitives/` и используют только токены (`var(--…)`), без дублирования «сырых» hex в модулях по возможности.
2. **Слой токенов:** палитра и семантические алиасы вынесены в `styles/tokens.css`, подключаются из `globals.css`. Семантика (`--color-text`, `--radius-md`, …) — предпочтительный интерфейс для новых компонентов. **Секции и сетка:** ритм вертикали/инлайна (`--section-padding-y`, `--section-padding-inline`, при необходимости `--section-padding-y-compact`), максимальная ширина контентной колонны как на главной (`--section-content-max: min(90rem, 100%)`), внутрисекционные зазоры и вертикальные поля карточек (`--section-inner-stack-gap`, `--section-card-padding-block`), смещение под фиксированный хедер (`--page-content-offset-top`) — единый контракт для CSS Modules; не дублировать «магические» `1440px` / `4vh` в новых модулях. **Плашка хедера без hero:** `--color-header-bar-bg` и `--color-header-bar-border` (палитра `brown`/`black`, не legacy `--blue`) — см. `widgets/layout/ui/header/Header.module.css` (`solidDark`). **Блок `Features` после `HeroBanner`:** внешние отступы секции = 0; визуальное «смыкание» с низом hero — через внутренний `features__container` (см. `shared/ui/features-section/ui/Features.module.css`), и на главной, и на внутренних лендингах.
3. **Storybook** — каноничный каталог визуальных контрактов для примитивов `shared/ui/primitives/**/*.stories.*`.
4. **NextUI / HeroUI:** не внедряются в этом ADR. Отдельное решение — только после оценки размера бандла, a11y и стоимости миграции; до этого момента **не добавлять** `@nextui-org/*` без нового ADR.

## Последствия

- Фичи и use-case слои **не импортируют** произвольный UI вне согласованных слоёв (`shared/`, `features/*/ui`, `widgets/` по правилам ESLint); корневого каталога `components/` в проекте больше нет — примитивы в `shared/ui/primitives/`.
- Постепенный перевод существующих блоков на токены — по мере касания файлов (incremental), без «большого взрыва».

## Проверка готовности

- `npm run storybook` / `npm run build-storybook` проходят.
- Новые UI-примитивы имеют stories и опираются на `styles/tokens.css`.
