# ADR 0002: Отказ от Tailwind и выбор Panda CSS

## Статус

Принято (2026-04-15)

## Контекст

- В бэклоге развития UI появился курс на Tailwind, но текущий проект уже стандартизован вокруг `CSS Modules` и централизованных токенов в `styles/tokens.css`.
- Для долгосрочной поддержки важны: типобезопасность стилей, предсказуемая связка с токенами, минимизация inline class-строк и контролируемая миграция без массовых регрессий.
- Команда хочет сохранить текущую архитектуру UI (примитивы + токены + Storybook), при этом получить современный utility/recipe-подход.
- Решение дополняет базовый контракт из ADR `0001-ui-stack-and-design-tokens.md`.

## Решение

1. **Tailwind не используется** для новых изменений и не добавляется в проект.
2. **Альтернатива:** использовать Panda CSS как utility/recipe слой поверх проектных токенов.
3. **Базовый контракт не меняется:** `CSS Modules + tokens` остаются рабочей базой; Panda CSS внедряется постепенно в новые или активно изменяемые компоненты.
4. **UI-компоненты:** сохраняется курс на собственные примитивы (`shared/ui/primitives/`) и headless-подход для сложного интерактива; тяжелые монолитные UI-kit подключаются только после отдельного ADR.

## Последствия

- Плюсы:
    - единый дизайн-контракт через токены;
    - типобезопасные recipes и предсказуемый DX;
    - нет резкой миграции всей кодовой базы.
- Компромиссы:
    - параллельное сосуществование `CSS Modules` и Panda CSS в переходный период;
    - потребуется дисциплина по границам: новые стили не должны дублировать токены и существующие примитивы.

## Проверка готовности

- Новый UI-код использует токены и не содержит raw hex вне `styles/tokens.css`.
- Для интерактива сохраняются требования a11y (`:focus-visible`, клавиатурная навигация, ARIA).
- Изменения отражаются в Storybook и в `docs/audit/audit-and-improvement-plan-ru.md`.
