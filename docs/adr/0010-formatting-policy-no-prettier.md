# ADR 0010: политика форматирования без Prettier

**Статус:** принято  
**Дата:** 2026-04-27

## Контекст

В бэклоге (`DEV-02`) обсуждалось добавление `yarn format`/`yarn format:check` на базе Prettier.
Текущий пайплайн уже включает `eslint --fix`, типизацию, unit и архитектурные гейты, а форматирование стабильно покрывается правилами ESLint.

## Решение

1. **Не добавлять Prettier** в текущем цикле.
2. Форматирование оставляем в зоне ответственности ESLint (`lint` / `lint:fix`) и code review.
3. Возвращаться к вопросу только при явной боли команды (частые конфликты стиля, рост времени ревью).

## Последствия

- Не расширяем стек инструментов и время CI.
- Избегаем дублирующихся правил форматирования (ESLint vs Prettier).
- Если решение поменяется, потребуется отдельный ADR с планом миграции (`eslint-config-prettier`, обновление hooks/CI/доков).

## Проверка готовности

- `package.json` не содержит `format`/`format:check`.
- `SCRIPTS_AND_QUALITY_GATES_RU.md` синхронизирован с принятым решением.
