# ADR 0006: Next.js API routes — same-origin и CORS

**Статус:** принято  
**Дата:** 2026-04-17

## Контекст

Публичные обработчики (`pages/api/*`) вызываются с того же origin, что и страницы сайта (формы → `POST /api/feedback`, служебные → `GET /api/sitemap`). Вопросы безопасности и CORS периодически поднимаются при аудите.

## Решение

- **Same-origin по умолчанию:** браузер не применяет CORS к «простым» same-origin запросам из документа того же сайта. Отдельные заголовки `Access-Control-Allow-Origin` для текущих сценариев **не задаём** — контракт «только наш фронт + серверные вызовы из того же приложения».
- **Если появится внешний клиент** (мобильное приложение, другой домен): не включать `*` с куками; явный allowlist origin, минимальный набор методов и заголовков, отдельная оценка CSRF и аутентификации.

## Последствия

- Документированное ожидание: публичные API не являются открытым B2B HTTP API для сторонних сайтов без отдельного ADR.

## Проверка готовности

- ADR связан с бэклогом §5.8; при появлении cross-origin клиента — новый ADR или дополнение.
