## Проверка после успешного `mine`

Выполняйте из каталога дворца, где лежит `mempalace.yaml` (или задайте рабочий каталог перед командами).

### 1. Статус

```powershell
cd $mp   # например C:\Users\user\Documents\mempalace-premier-design
python -m mempalace status
```

**Ожидание:** ненулевое число drawers, комнаты (`ROOM`) согласованы с ожидаемым объёмом файлов.

### 2. Семантический поиск

```powershell
$env:PYTHONIOENCODING = 'utf-8'
$env:PYTHONUTF8 = '1'
python -m mempalace search "ваш запрос на естественном языке" --results 5
```

**Ожидание:** список результатов с указанием исходного файла и фрагмента текста, релевантного запросу.

**Windows:** без UTF-8 команда `search` может завершиться с `UnicodeEncodeError` из‑за символов рамки в выводе — задайте переменные окружения, как выше (или `chcp 65001` в `cmd`). Имеет смысл прописать их в профиле PowerShell, если пользуетесь MemPalace часто.

**Сужение выдачи:** `python -m mempalace search "запрос" --wing <имя> --room <имя>` — см. `python -m mempalace search --help`.

### 3. Wake-up (контекст для промпта)

```powershell
python -m mempalace wake-up
```

**Ожидание:** короткий блок контекста верхнего уровня (L0/L1 в терминологии MemPalace) для вставки в диалог с моделью. Для конкретного wing: `python -m mempalace wake-up --wing <имя>`.

### 4. MCP в Cursor

После настройки `mcp.json` (см. [`../cursor/mcp.mempalace.example.json`](../cursor/mcp.mempalace.example.json)) инструменты MemPalace должны быть видны в списке MCP. Перезапуск Cursor обязателен после изменения конфига.

**Проверка вручную:** в терминале выполните `python -m mempalace.mcp_server --palace "<полный путь к $mp>"` — процесс должен ждать stdio без немедленного падения (остановите Ctrl+C).

## Типовой цикл работы

1. Обновили материалы в дворце (скопировали новые `.md` из [`rules/`](rules/) репозитория или свои заметки).
2. `python -m mempalace mine $mp --dry-run` — убедиться, что список файлов ожидаем.
3. `python -m mempalace mine $mp` — переиндексация.
4. `python -m mempalace search "..."` — проверка выдачи.

Повторный `mine` по тем же файлам: MemPalace помечает уже обработанные записи (`Files skipped (already filed)` при повторе — нормально).

## Обслуживание индекса

| Задача                                                       | Команда (ориентир)                                         |
| ------------------------------------------------------------ | ---------------------------------------------------------- |
| Подозрение на повреждение векторов                           | `python -m mempalace repair` (см. `--help` в вашей версии) |
| Сжатие накопленных «ящиков» для экономии токенов в контексте | `python -m mempalace compress`                             |
| Статус ингеста                                               | `python -m mempalace status`                               |

Точные флаги и поведение — в официальной справке MemPalace для установленной версии.

## Связь с репозиторием Premier Design

- **Источник правил в git:** каталог [`rules/`](rules/) вместе с гайдами в этой папке (`mempalace-*-ru.md`).
- **Дворец MemPalace** — локально вне git: копируйте в `$mp` нужные файлы из `docs/mempalace/`, затем `mine`. Так в индекс не попадают секреты и лишний объём репозитория.

## Если что-то не так

| Симптом                       | Действие                                                                                                                                                       |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `No mempalace.yaml`           | `mine` запускать с корнем проекта после `init`, см. [`mempalace-agent-memory-ru.md`](mempalace-agent-memory-ru.md).                                            |
| Поиск пустой                  | Проверить, что запрос соответствует содержимому; увеличить `--results`; выполнить `status`; убедиться, что после копирования новых файлов был выполнен `mine`. |
| Краш при выводе в консоль     | UTF-8 для Python, см. раздел «Семантический поиск».                                                                                                            |
| MCP не отвечает               | Проверить `python -m mempalace.mcp_server --palace <путь>` вручную; путь к `python` в `mcp.json` совпадает с тем, где установлен MemPalace.                    |
| Устаревший контекст в ответах | Переиндексировать (`mine`) после обновления файлов в `$mp`.                                                                                                    |
