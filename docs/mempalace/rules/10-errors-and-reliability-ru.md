# Ошибки, надёжность и наблюдаемость

Разделение **ожидаемых** и **неожиданных** сбоев — основа предсказуемого UX и безопасного API.

## Ожидаемые сбои

- Валидация входа, 404, конфликт бизнес-правила — **явные коды и сообщения** для клиента без внутренних деталей.
- Пользователю — что исправить; в лог — без PII и секретов.

## Неожиданные сбои

- Сеть, БД, таймаут внешнего API — **логирование** с контекстом запроса (id, маршрут), ответ клиенту обобщённый (500 / service unavailable по политике).
- **Не** отдавать клиенту stack trace продакшена.

## Таймауты и идемпотентность

- Внешние вызовы — осмысленные **таймауты** там, где поддерживает библиотека.
- Формы и платежи — учитывать **двойной сабмит** и повтор запроса (идемпотентные ключи, rate limit не обходить).

## Циклические зависимости

- Запрещены; при риске — вынести общее в `shared/` или узкий модуль с однозначным именем.

## Критерии готовности

- Новый API-роут имеет таблицу «код → смысл» в коде или в соседнем тесте.
- Ошибка в логах позволяет найти причину за один проход по correlation id / request id, если принято в проекте.
