# Чистый код: SOLID, DRY, KISS, YAGNI (прагматично)

Согласовано с регламентом проекта: без «архитектуры на вырост» и без священных абстракций.

## SOLID (сжато)

- **SRP:** один модуль — одна зона ответственности; не смешивать формат ответа API и расчёт цены в одном файле.
- **OCP:** расширять новыми функциями/вариантами, а не бесконечным `switch`; фабрики «на всё будущее» — только при реальной потребности.
- **LSP / ISP / DIP:** интерфейсы — когда есть **вторая** реализация или тестовый дублёр; иначе YAGNI.

## DRY

- Не дублировать код; **три копипасты** с расхождением — сигнал к выносу в `shared/` или узкий util с **говорящим именем**.
- Дублирование хуже преждевременной абстракции, пока копипаст не **доказан** расхождениями.

## KISS

- Ранние возвраты, плоский поток чтения, без «магических» флагов состояния.
- Имена сущностей **самодокументируемые**; без венгерской нотации и нерасшифрованных аббревиатур домена.

## YAGNI

- Нет плагин-систем, конфигурации «на год вперёд» и абстрактных репозиториев без второго хранилища — пока продукт явно не запросил.

## Критерии готовности

- Новый код читается без контекста «что автор имел в виду».
- Нет закомментированного мёртвого кода «на всякий случай» без задачи на удаление или восстановление.
