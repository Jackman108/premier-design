# Типизация, контракты и валидация

Явные контракты снижают регрессии при изменении границ системы.

## TypeScript

- Публичные функции, экспорты из `features/*/`, API-хелперы — **явные сигнатуры**; возвращаемые типы осмысленны, не «просто object».
- **`any`** — запрещён без комментария «почему» и пути к типобезопасной замене (если так требует политика репозитория).

## Валидация границ

- **Query, body, env** — валидация схемой (например Zod) **до** использования в логике.
- Невалидный ввод — ранний отказ с понятной ошибкой, без частичного выполнения сценария.

## Контракты API

- Версионирование или явная эволюция полей; breaking changes — через документацию и тесты `tests/api/*`, если принято в проекте.

## Критерии готовности

- Новые поля формы / query проходят через одну схему валидации, общую с сервером там, где это возможно (shared validates).
