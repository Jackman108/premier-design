# Доступность (a11y) и UX

Минимальный стандарт для публичных веб-интерфейсов и форм.

## Семантика и структура

- Использовать **нативные элементы** там, где они решают задачу: `button`, `a`, `label`, `input`, заголовки уровней `h1–h6` по смыслу документа.
- **Один главный `h1`** на логическую страницу (включая SPA-страницы с маршрутизацией).
- Ландмарки (`main`, `nav`, `header`, `footer`) — когда улучшают ориентацию, без избыточной вложенности.

## Формы

- У каждого поля — **`label`**, связанный с `id`/`htmlFor` (или эквивалент для кастомных контролов — `aria-labelledby` / `aria-label` с осмысленным текстом).
- Сообщения об ошибках — **рядом с полем** и/или в `aria-describedby`; фокус при отправке с ошибками — на первое невалидное поле.
- Не полагаться только на цвет для состояния ошибки (иконка + текст).

## Клавиатура и фокус

- Весь интерактив доступен с **клавиатуры**; порядок таба логичен (без `tabindex` > 0 без крайней нужды).
- **Видимый focus ring** (не `outline: none` без замены контрастной обводкой).
- Модальные окна: ловушка фокуса, закрытие по `Escape`, возврат фокуса на триггер.

## Моторика и вестибулярная чувствительность

- Уважать **`prefers-reduced-motion`**: отключать или сильно сокращать декоративные анимации.
- Не использовать автоплей видео со звуком; для каруселей — пауза и явное управление.

## Критерии готовности

- Пройден ручной чеклист: таб по форме, отправка с ошибками, screen reader spot-check для ключевого сценария.
- Контраст текста и интерактивов — не ниже **WCAG 2.1 AA** для типового UI (или задокументированное исключение с дизайн-обоснованием).
